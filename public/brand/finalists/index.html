Jingle brand finalists

三条入围方向的真实落地感

G-05、H-01、F-04 单独展开。这里不再继续抽卡，而是看它们在桌面 app、Web 标志、favicon、小尺寸、light / dark、flat、全貌 lockup、衣服和周边上的气质差异。

G-05 · Portal

最像“万能入口”。品牌故事强，适合讲工具被召唤出来。

H-01 · Chime

最极简。记忆点是声音和光点，更适合长期品牌符号。

F-04 · Command

最产品化。打开就知道是 launcher / command 工具。

G-05 · Clean Portal

入口感最强，像一个被唤醒的工具门

这个方向把“万能口袋”翻译成一个干净的传送门。它不直接画口袋，也不画工具列表，而是把核心体验压成一个动作：入口打开，工具出现。适合做有故事感的品牌主标。

品牌故事强 App icon 有存在感 Web 端偏神秘

Desktop app

在 dock 里最像一个“入口”，不依赖文字也能成立。

Favicon / icon

Light / dark

深色里最自然；浅色里也能靠深底 icon 自洽。

Flat system

扁平后像“门洞 + 唤醒光”，可用于单色系统图标。

Full lockup

Jingle金狗 · 万能入口

适合官网 hero、启动页、发布图，故事完整。

Apparel

印在衣服上像一个徽章，品牌感强但稍偏科技神秘。

Merch

适合贴纸和胸针，像“入口徽章”。

强项

最能承接“万能口袋”的想象，但没有直接复刻任何具体角色。

风险

如果产品想更工具化，它可能稍微偏奇幻，Web B2B 语境要收一下。

适合

品牌主图、启动页、社区头像、发布物料。

H-01 · Chime Dot

最轻，像“叮”的声音被看见

H-01 把品牌压到最小：一个声音点，一道信号弧。它不急着解释产品功能，而是建立一个可以长期重复的识别符号。越小越干净，越适合 favicon、菜单栏、状态点和贴纸。

最极简 小尺寸最好 品牌延展最灵活

Desktop app

在 dock 里很安静，像一个被点亮的动作按钮。

Web logo

JingleTools appear at the chime

配文字后很现代，留白多，适合官网和开发者文档。

Favicon / icon

16px 还保留“亮点”，是三个方案里小尺寸最稳的。

Light / dark

深浅环境都不抢，适合放在导航、托盘和状态位。

Flat system

扁平后只剩“音点 + 弧”，可以变成一套系统符号。

Full lockup

Jingle金狗 · 叮一下

全貌很轻，适合长期主品牌，但产品功能感要靠文案补。

Apparel

在衣服上更像潮牌小标，低调但识别点明确。

Merch

周边感最好，像一个声音印章，容易做贴纸和 pin。

强项

极简、耐看、小尺寸稳，最容易从 app icon 扩展到系统符号。

风险

第一眼不直接说明“效率工具”，需要 Jingle 的产品语言一起建立。

适合

favicon、菜单栏、状态点、品牌水印、开发者页导航。

F-04 · Command Capsule

最像产品本身，输入框就是入口

F-04 直接把 launcher / command surface 抽成一个胶囊。它是三个方案里最容易让用户理解产品用途的一个：打开、输入、找到工具、执行。气质更产品化，也更适合官网和桌面工具定位。

功能识别最快 Web 端最稳 更像工具品牌

Desktop app

在 dock 里能直接读成命令入口，产品属性最强。

Web logo

JingleCommand everything from one place

Web 导航最清楚，像一个成熟效率软件的主标。

Favicon / icon

64 和 32 很清晰；16px 会变成一个发光胶囊，细节会损失。

Light / dark

深浅都可用，尤其适合官网产品图和应用商店图。

Flat system

扁平后仍然很产品化，可以直接成为功能图标。

Full lockup

Jingle金狗 · 命令入口

官网和开发者文档最稳，第一眼就是效率工具。

Apparel

衣服上更像产品团队周边，不如 H-01 那么抽象潮牌。

Merch

贴纸可用，但更像软件 icon，不像独立符号。

强项

产品识别最快，官网、文档、应用图标都很稳。

风险

品牌想象力比 G-05、H-01 少一些，容易和普通搜索/命令工具靠近。

适合

官网主标、应用商店、桌面 app、开发者平台。
