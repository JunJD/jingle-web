Jingle 的 AI 方向不是再放一个聊天框，而是让 AI 能在用户可见、可审批、可回放的边界内使用扩展工具、本地上下文和用户明确配置的个人偏好。

<Callout title="可用范围" tone="warning">
  Agent 工具调用已进入预览阶段。AI Commands、Quick AI、上下文选择、参数编辑、审计日志、失败恢复和 eval 会按平台路线逐步开放。
</Callout>

## 平台边界

- Agent 不应该直接拿到 access token、refresh token 或 client secret。
- Agent 可以看到工具描述、连接状态、输入参数和审批结果。
- AI profile / personalization 属于用户可编辑设置，不应变成不可见的隐式画像。
- 扩展工具在 main service 执行，宿主负责权限、连接解析和结果展示。
- 对会修改外部系统的工具，审批和参数展示必须比“单纯确认按钮”更细。

## 建设方向

1. 稳定工具注册、连接状态和展示元数据。
2. 给修改型工具补 schema form 审批和参数编辑。
3. 给 GitHub、Notion 等 provider 工具补端到端 eval。
4. 把 AI Command 从 prompt 变成可复用、可发布、可共享的命令单元。
5. 让 Quick AI 和 AI Profile 在产品上可见、可关闭、可编辑。
