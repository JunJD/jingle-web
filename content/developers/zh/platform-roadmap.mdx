<Callout title="发布阶段" tone="warning">
  “预览可用”表示能力可用于私有扩展或早期集成；“正在研发中...”表示平台正在建设完整产品路径；“规划中”表示方向明确，但 API、体验和发布时间仍会随版本演进。
</Callout>

## 开发者边界

- 扩展作者可以依赖已标记为“预览可用”的 package、runtime SDK、账号连接和 Agent 工具能力。
- OAuth、公开商店、签名更新和大规模第三方发布会由平台提供统一流程。
- 账号、OAuth、团队、商店和同步相关能力由平台管理安全边界，扩展不直接保存 OAuth token 或 refresh token。
- Jingle 当前路线图聚焦桌面端工作流；跨设备同步以桌面平台和未来明确支持的端为准。

## 近期优先级

1. 先把扩展包、运行时 SDK、账号连接和 Agent 工具的边界稳定下来。
2. 再补 OAuth 平台服务和标准 Connect UI，让 GitHub、Notion 等公开 provider 可以完成真正授权闭环。
3. 接着建设扩展分发：模板、打包校验、私有分发、公开目录和审核流程。
4. 最后逐步扩展到团队共享、片段/剪贴板、窗口管理、同步和治理体系。
