## 核心概念

| 概念 | 作用 | 状态 |
| --- | --- | --- |
| Extension package | 扩展的分发单元，包含 manifest、runtime、main service、assets 和工具定义 | <StatusBadge tone="preview">预览可用</StatusBadge> |
| Runtime SDK | 用 React 组件构建列表、详情、表单、动作面板、菜单栏入口和反馈 | <StatusBadge tone="preview">预览可用</StatusBadge> |
| Connected accounts | 由宿主管理账号连接、凭证读取、连接状态和设置页跳转 | <StatusBadge tone="preview">预览可用</StatusBadge> |
| OAuth / PKCE | 平台托管的浏览器授权、callback、token exchange、refresh 和 revoke | <StatusBadge tone="research">规划中</StatusBadge> |
| Agent tools | 让 Agent 在用户可见、可审批的边界内调用扩展能力 | <StatusBadge tone="preview">预览可用</StatusBadge> |

## 平台范围

Jingle Developers 聚焦扩展作者需要接触的部分：扩展包、运行时 API、账号连接、OAuth、Agent 工具和分发流程。启动器、文件搜索、片段、剪贴板、窗口管理、团队同步和安全治理属于产品平台路线图，并会在对应文档中说明开发者可依赖的边界。

查看 [平台路线图](/zh/developers/platform-roadmap)，了解每个能力面的阶段安排。

- 需要 Agent 调用扩展能力时，阅读 [Agent 工具](/zh/developers/agent-tools)。

## 一个最小命令

扩展命令使用 React 描述桌面 UI，并通过 Jingle host 执行动作：

```tsx
import { Action, ActionPanel, List, showToast } from "@openwork/extension-api"

export default function Command() {
  return (
    <List>
      <List.Item
        title="Run workflow"
        actions={
          <ActionPanel>
            <Action title="Confirm" onAction={() => showToast({ title: "Done" })} />
          </ActionPanel>
        }
      />
    </List>
  )
}
```

命令代码不需要自己维护窗口、设置页或凭证存储。扩展声明所需能力，Jingle 负责把运行时、账号连接和用户操作串起来。
