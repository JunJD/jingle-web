Jingle Extension API gives commands UI components, host capabilities, and runtime utilities. During Developer Preview, the package name is `@openwork/extension-api`; public release naming will be unified with the platform release cadence.

<Callout title="API stability" tone="warning">
  This page documents the API available during Developer Preview. “Preview” means the capability has a runtime implementation and can be used in private extensions and early integrations; names, parameters, failure semantics, and compatibility policy will continue to stabilize before public release.
</Callout>

## Preferences

### `getPreferenceValues()`

Reads extension-level and command-level settings resolved for the current launch context.

```ts
const preferences = getPreferenceValues<{
  apiBaseUrl: string
  open_in: { name?: string; bundleId?: string; path?: string }
}>()
```

### `getConnectionSecret(name)`

Reads a host-managed connection secret. Extensions request the secret names they need; the platform owns where credentials are stored.

```ts
const token = getConnectionSecret("accessToken")
```

### `openExtensionPreferences()` / `openCommandPreferences()`

Opens the extension or command settings screen. When a command needs configuration or a connected account, it can hand off to platform settings instead of implementing its own setup UI.

```tsx
import { Action, ActionPanel, Detail, openExtensionPreferences } from "@openwork/extension-api"

export default function Command() {
  return (
    <Detail
      markdown="Connect this extension before running the command."
      actions={
        <ActionPanel>
          <Action title="Open Extension Settings" onAction={openExtensionPreferences} />
        </ActionPanel>
      }
    />
  )
}
```

| API | Semantics | Status |
| --- | --- | --- |
| `LocalStorage.getItem` | Read an extension-scoped persisted value | <StatusBadge tone="preview">Preview</StatusBadge> |

| `LocalStorage.removeItem` | Remove an extension-scoped value | <StatusBadge tone="preview">Preview</StatusBadge> |
| `LocalStorage.clear` | Clear extension-scoped storage | <StatusBadge tone="preview">Preview</StatusBadge> |
| `LocalStorage.allItems` | Read all extension-scoped values | <StatusBadge tone="preview">Preview</StatusBadge> |
| `Cache` | Synchronous namespaced cache with subscriptions and capacity trimming | <StatusBadge tone="preview">Preview</StatusBadge> |

<Callout title="Sensitive data boundary" tone="warning">
  LocalStorage and Cache are for extension state. Do not store access tokens, refresh tokens, or client secrets there. Account credentials belong to connection secrets / credential storage.
</Callout>

## Utilities

`@openwork/extension-utils` provides connected-account helpers for token-backed integrations:

| API | Semantics |
| --- | --- |
| `OAuthService.getAccessToken()` | Reads `accessToken` from a host-managed connection secret |
| `OAuthService.authorize()` | Reads the current connection token; see OAuth for browser authorization |
| `withAccessToken(service)` | Shows a connection empty view for React commands; throws for non-UI functions |
| `getAccessToken(service)` | Reads the current connection secret or service token |

<Callout title="Migrating to platform OAuth" tone="info">
  Token-backed integrations are useful for Developer Preview and private workflows. Public third-party integrations should migrate to platform-owned OAuth so Jingle can manage authorization, refresh, revoke, and credential storage.
</Callout>
